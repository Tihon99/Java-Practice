Практика 15: Логические блоки

В этой практике демонстрируется порядок выполнения логических блоков, статических блоков и конструктора при создании объекта класса.

Файл A.java содержит:
1. Два нестатических логических блока
2. Один статический логический блок
3. Поле id
4. Конструктор

Порядок выполнения при создании объекта new A(100):

1. Статический блок:
   - Выполняется первым и только один раз при первом обращении к классу
   - Выводит: "static logic"

2. Инициализация полей значениями по умолчанию:
   - id получает значение 0

3. Первый логический блок:
   - Выполняется после инициализации полей по умолчанию
   - Выводит: "logic (1) id= 0"
   - id все еще равен 0, так как явная инициализация еще не произошла

4. Инициализация поля явным значением:
   - id = 1

5. Второй логический блок:
   - Выполняется после явной инициализации полей
   - Выводит: "logic (2) id= 1"
   - id уже равен 1 из явной инициализации

6. Конструктор:
   - Выполняется последним
   - Устанавливает id = 100
   - Выводит: "ctor id= 100"

Итоговый вывод программы:
static logic
logic (1) id= 0
logic (2) id= 1
ctor id= 100

Важные замечания:
1. Статический блок выполняется только один раз при первом обращении к классу
2. Нестатические логические блоки выполняются при каждом создании объекта
3. Логические блоки выполняются в том порядке, в котором они определены в классе
4. Конструктор всегда выполняется последним
5. Логические блоки не наследуются подклассами
