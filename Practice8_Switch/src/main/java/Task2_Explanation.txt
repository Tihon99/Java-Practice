Когда в case отсутствует break, выполнение кода "проваливается" в следующий case без проверки его условия.

В первом примере (value = 2):
1. Управление переходит к case 2
2. Выполняется i = 2
3. Из-за отсутствия break, выполнение "проваливается" в case 3
4. Выполняется i = 3
5. Встречается break, выполнение switch завершается
Поэтому итоговое значение i = 3

Во втором примере (value = 1):
1. Управление переходит к case 1
2. Выполняется i = 1
3. Встречается break, выполнение switch завершается
Поэтому итоговое значение i = 1
